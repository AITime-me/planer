# Список задач

Простое веб-приложение для планирования задач. Работает в браузере, данные сохраняются локально.

## Возможности

### Две вкладки

- **Текущие задачи** — задачи по дням недели, датам и времени
- **План на будущее** — задачи с указанием дедлайна

### Поля задачи

| Поле | Описание |
|------|----------|
| Название | Текст задачи |
| Дата | Дата выполнения или дедлайн |
| День недели | Определяется автоматически по дате |
| Время «с» / «до» | Необязательный интервал выполнения |
| Приоритет | Цветовая метка по матрице Эйзенхауэра |

### Приоритеты

| Цвет | Значение |
|------|----------|
| Красный | Важно и срочно |
| Зелёный | Важно, но не срочно |
| Жёлтый | Не важно и не срочно |
| Синий | Срочно, но не важно |

### Темы оформления

Светлая, тёмная, бежевая, оливковая, синяя — переключатель внизу экрана.

## Быстрый старт

### Вариант 1: открыть файл

Дважды кликните по `index.html` или перетащите файл в окно браузера.

### Вариант 2: локальный сервер (рекомендуется)

```bash
npm start
```

Приложение откроется по адресу [http://localhost:3000](http://localhost:3000).

> Для `npm start` нужен [Node.js](https://nodejs.org/). Устанавливать зависимости не требуется — используется `npx serve`.

## Хранение данных

Задачи и выбранная тема сохраняются в `localStorage` браузера. Данные остаются после перезагрузки страницы и не отправляются на сервер.

## Git и публикация

### Инициализация репозитория

```bash
git init
git add .
git commit -m "Initial commit: приложение «Список задач»"
```

### GitHub Pages

В репозитории есть workflow `.github/workflows/pages.yml`. После push в ветку `main` или `master`:

1. Откройте **Settings → Pages** в репозитории на GitHub
2. В разделе **Build and deployment** выберите источник **GitHub Actions**
3. Дождитесь завершения workflow — приложение будет доступно по адресу вида `https://<username>.github.io/<repo>/`
